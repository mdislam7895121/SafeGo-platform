# SAFEGO GLOBAL MASTER RULES (MUST FOLLOW)
###############################################

1. SYSTEM STRUCTURE RULES  
- SafeGo-এর ৪টি রোল সবসময় সক্রিয় থাকবে: Customer, Driver, Restaurant, Admin.  
- প্রতিটি রোলের স্ক্রিন, লজিক, স্ট্যাটাস ফ্লো স্বাধীনভাবে কাজ করবে।  
- Web + Mobile future-ready architecture বজায় থাকবে।  
- স্টোর (rides, deliveries, food_orders) সবসময় সঠিকভাবে আপডেট হবে।

2. COUNTRY RULES  
- USA mode: No cash payments. শুধু Card, Wallet, Apple Pay (Coming soon).  
- GPS auto-detect সবসময় অন.  
- Payment method manually choose করা যাবে না যদি backend incomplete থাকে → show “Coming soon”.

3. SECURITY RULES  
- Sensitive flow-তে input validation, data sanitization, server-side fare calculation.  
- Fare user কখনও manually input দিতে পারবে না।  
- Locations, prices, payments — সব backend থেকে আসবে।  
- Unauthorized access → auto redirect.

4. STATUS FLOW RULES  
- Customer→Ride Request→Driver Accept→On-the-way→Arrived→Trip Started→Completed.  
- Courier flow, Food flow — future steps হিসেবে placeholder থাকবে।  
- সব UI elements-এ data-testid থাকতে হবে।

5. DESIGN RULES  
- পুরো Customer Web App Uber-এর মতো ultra-clean UI হবে।  
- Mobile-first responsive layout.  
- Everything swipeable, scroll-friendly.  
- কোনো কিছু unfinished থাকলে → “Coming soon” toast দেখাবে।

6. DEVELOPMENT RULES  
- এক স্ক্রিন শেষ করে লগ করতে হবে।  
- Broken link কখনও থাকবে না।  
- Bottom Tabs show হবে, কিন্তু ফিচার incomplete হলে Coming soon.  
- Speed optimized, no large scripts.  
- UX সর্বোচ্চ লেভেলের হবে।
